# 词首/词末/对语 查询库设计方案

## 1. 目标

实现 PRD 中 F4（底部多功能字典）描述的三项核心功能：

| 功能 | 触发条件 | 说明 |
|------|----------|------|
| **词首** | 输入 1 个字 | 查找以该字**开头**的 2/3/4 字诗词用语，按**邻字平仄**分桶 |
| **词末** | 输入 1 个字 | 查找以该字**结尾**的 2/3/4 字诗词用语，按**邻字平仄**分桶 |
| **对语** | 输入 1~4 个字 | 查找在对仗中与该词**同位置对应**的词语 |

## 2. 数据来源

基于项目中 4 份带 metadata 的诗词集统计生成：

| 文件 | 内容 | 数量 |
|------|------|------|
| `poems_with_metadata.json` | 今人诗词 | 3,081 首 |
| `tang_poems_with_metadata.json` | 唐诗 | 29,904 首 |
| `song_poems_with_metadata.json` | 宋诗 | 142,964 首 |
| `song_ci_with_metadata.json` | 宋词 | 9,924 首 |
| **合计** | | **185,873 首** |

各功能使用的数据范围不同：

| 功能 | 数据范围 | 说明 |
|------|----------|------|
| 词首/词末 | 全部 185,873 首（1,213,839 行） | 越广越好，覆盖所有诗词用语 |
| 对语 | 仅律诗 95,992 首（191,984 个行对） | 只有律诗的颔联/颈联有严格对仗要求 |

## 3. 核心算法

### 3.1 分句

将每首诗词的 `content` 字段按中文标点分割，提取纯汉字行（长度 >= 2）：

```
"明月松间照，清泉石上流。" → ["明月松间照", "清泉石上流"]
```

### 3.2 词首/词末提取（含邻字平仄分桶）

对每一行，提取行首和行末的 2/3/4-gram，并根据**邻字**的平仄归入 P 桶或 Z 桶：

**邻字定义**：
- **词首**：邻字 = n-gram 的第 2 个字（紧跟查询字之后）
- **词末**：邻字 = n-gram 的倒数第 2 个字（紧邻查询字之前）

```
查 "花" 词首:
  "花开"  → 邻字"开"=平 → P桶
  "花落"  → 邻字"落"=仄 → Z桶
  "花间"  → 邻字"间"=可平可仄 → P桶 + Z桶 (两桶都放)

查 "花" 词末:
  "梅花"  → 邻字"梅"=平 → P桶
  "落花"  → 邻字"落"=仄 → Z桶
  "杏花"  → 邻字"杏"=仄 → Z桶
```

**平仄判断**依赖 `char_dict.json`（构建时预加载）。判断规则与 checker 一致：
- 声调含"平/阴平/阳平" → 平声 (P)
- 声调含"上/去/入" → 仄声 (Z)
- 兼有平仄声调 → 可平可仄 (A)，**归入 P 和 Z 两个桶**
- 查无此字 → 视为可平可仄，归入两桶

以首字（或末字）为索引 key，按 n-gram 长度和邻字平仄两个维度分组，统计全库出现频次。

### 3.3 对语（对仗配对）提取

#### 来源筛选

仅从**律诗**中提取对仗数据。筛选条件：诗词 metadata 中 `closest_rule` 字段以 **"五律"** 或 **"七律"** 开头。全库符合条件的律诗分布：

| 来源 | 律诗数量 |
|------|----------|
| 唐诗 | 17,949 首 |
| 宋诗 | 77,072 首 |
| 今人诗词 | 971 首 |
| 宋词 | 0 首（词无律诗） |
| **合计** | **95,992 首** |

#### 行对提取：仅颔联与颈联

律诗 8 句 4 联的结构中，只有中间两联（颔联、颈联）有严格的对仗要求：

```
第1句 ┐
第2句 ┘ 首联 —— 不要求对仗，跳过
第3句 ┐
第4句 ┘ 颔联 —— 必须对仗 ✓ 提取
第5句 ┐
第6句 ┘ 颈联 —— 必须对仗 ✓ 提取
第7句 ┐
第8句 ┘ 尾联 —— 不要求对仗，跳过
```

每首律诗贡献 **2 个行对**（颔联 1 对 + 颈联 1 对），共得到 **191,984 个行对**。

#### n-gram 对提取

对每个行对，在所有合法起始位置提取同位置、同长度的 n-gram 对（n = 1, 2, 3, 4）：

```
颔联上句: 明月松间照
颔联下句: 清泉石上流

位置0, n=1: "明" ↔ "清"
位置0, n=2: "明月" ↔ "清泉"
位置2, n=2: "松间" ↔ "石上"
位置4, n=1: "照" ↔ "流"
...共 14 组 n-gram 对 (5言)
```

**双向记录**：每个对应关系双向存储（"明月"→"清泉" 和 "清泉"→"明月"），使得无论用户输入哪一侧都能查到对方。

**去噪机制**：未做显式分词，依靠**统计频率自然过滤**——跨越词边界的 n-gram（如"月松"）在全库中极少重复出现，会被最低频次阈值过滤掉；真正有意义的对仗词组（如"明月"↔"清泉"）在大量诗作中反复出现，自然高频留存。

### 3.4 阈值与裁剪

| 参数 | 词首/词末 | 对语 |
|------|-----------|------|
| 最低频次 | >= 2 | 1字: >= 3, 2字: >= 3, 3字: >= 5, 4字: >= 5 |
| 每个 key **每桶** 保留 | Top 50 | Top 50 |

词首/词末的 Top 50 是**按 P/Z 桶分别计算**的。以 2 字词为例，每个查询字最多保留 50(P) + 50(Z) = 100 条结果（可平可仄的词会同时出现在两桶中）。

对语对较长 n-gram 使用更高阈值，因为 3/4 字的精确对仗在不同诗作中重复概率低，低频条目多为跨词边界噪音。

## 4. 输出文件

构建脚本 `build_phrase_db.py` 一次性生成 3 个 JSON 文件，存放在 `static/config/`：

| 文件 | 大小 | 条目数 | 说明 |
|------|------|--------|------|
| `phrase_head.json` | 3.8 MB | 4,694 个首字 | 词首索引，按邻字平仄分桶 |
| `phrase_tail.json` | 4.4 MB | 4,666 个末字 | 词末索引，按邻字平仄分桶 |
| `phrase_pairs.json` | 3.3 MB | 85,747 个词条 | 对语索引（仅律诗颔联+颈联） |
| **合计** | **11.4 MB** | | |

### 4.1 phrase_head.json 结构

```json
{
  "花": {
    "2": {
      "P": [["花开", 253], ["花前", 116], ["花间", 113], ...],
      "Z": [["花落", 241], ["花间", 113], ["花下", 99], ...]
    },
    "3": {
      "P": [["花开花", 45], ["花开不", 11], ...],
      "Z": [["花落花", 36], ["花落春", 14], ...]
    },
    "4": {
      "P": [...],
      "Z": [...]
    }
  }
}
```

key 为首字，第一层按 n-gram 长度（"2"/"3"/"4"）分组，第二层按邻字平仄（"P"/"Z"）分桶，每桶为 `[词语, 出现次数]` 的数组，已按频次降序排列。

注意："花间"同时出现在 P 和 Z 两桶中，因为"间"字可平可仄。

### 4.2 phrase_tail.json 结构

与 phrase_head.json 结构完全对称，key 为末字，邻字为倒数第 2 个字：

```json
{
  "花": {
    "2": {
      "P": [["梅花", 342], ["桃花", 160], ["黄花", 118], ...],
      "Z": [["落花", 172], ["杏花", 78], ["树花", 72], ...]
    },
    "3": {
      "P": [["后庭花", 38], ["看梅花", 33], ...],
      "Z": [["后庭花", 38], ["不见花", 23], ...]
    }
  }
}
```

### 4.3 phrase_pairs.json 结构

（无变化）key 为查询词（1~4 字），value 为 `[对应词, 出现次数]` 数组：

```json
{
  "花": [["叶", 810], ["草", 568], ["柳", 481], ["水", 439], ...],
  "明月": [["白云", 63], ["清风", 39], ["夕阳", 20], ...],
  "天地": [["江湖", 57], ["山川", 27], ["江山", 24], ...]
}
```

查询词与结果等长（查 1 字返回 1 字对、查 2 字返回 2 字对），这由构建时"同位置同长度提取"保证。

## 5. API 设计

对应 PRD 中 `GET /api/dictionary/search`：

```
GET /api/dictionary/search?term=花&mode=head&length=2&tone=P
GET /api/dictionary/search?term=花&mode=tail&length=3&tone=Z
GET /api/dictionary/search?term=明月&mode=pair
```

| 参数 | 说明 |
|------|------|
| `term` | 查询词（1~4 个字） |
| `mode` | `head`（词首）/ `tail`（词末）/ `pair`（对语） |
| `length` | 仅 head/tail 有效，可选 `2` / `3` / `4` / `all` |
| `tone` | 仅 head/tail 有效，按邻字平仄筛选，可选 `P`（平）/ `Z`（仄）/ `all`（不筛选，合并两桶） |

**后端实现**（伪代码）：

```python
def search_dictionary(term, mode, length='all', tone='all'):
    if mode == 'head':
        data = phrase_head.get(term, {})  # term 必须是单字
        if length != 'all':
            length_data = data.get(str(length), {})
            if tone != 'all':
                return length_data.get(tone, [])
            # 合并 P+Z 桶，按频次重新排序
            merged = {}
            for t in ['P', 'Z']:
                for item in length_data.get(t, []):
                    merged[item[0]] = max(merged.get(item[0], 0), item[1])
            return sorted(merged.items(), key=lambda x: -x[1])
        return data  # 返回所有长度+平仄的完整结构
    
    elif mode == 'tail':
        # 同 head 逻辑
        ...
    
    elif mode == 'pair':
        return phrase_pairs.get(term, [])  # term 可为 1~4 字
```

## 6. 前端交互流程

```
用户在字典输入框键入文字
        │
        ▼
    字数判断
    ┌───────────────────────────────────┐
    │ == 1 字                           │ > 1 字
    │ 显示 4 个 Tab:                    │ 仅显示 [对语] Tab
    │ [韵部] [词首] [词末] [对语]        │
    └───────────────────────────────────┘
        │                                    │
   用户选 [词首] 或 [词末]                    │
        │                                    │
        ▼                                    ▼
   第三行: 两组垂直筛选器              直接展示对语结果
   ┌─────────────────────────┐
   │ 字数: ●◦  ●◦◦  ●◦◦◦    │ (词首)
   │       ◦●  ◦◦●  ◦◦◦●    │ (词末)
   │ 平仄: [平] [仄] [全部]  │ ← 新增
   └─────────────────────────┘
        │
        ▼
   第四区域展示结果列表
   (按频次降序，显示词语和频次)
```

## 8. 构建方法

```bash
cd /path/to/shiva
python build_phrase_db.py
```

耗时约 20 秒（需预先有 `static/config/char_dict.json` 用于邻字平仄判断）。输出至 `static/config/` 目录。无需额外依赖，仅用 Python 标准库。
