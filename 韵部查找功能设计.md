# 韵部查找功能设计：正向查找与逆向查找

## 1. 现有数据资产概览

项目中有两个核心 JSON 数据文件，分别支撑正向查找和逆向查找。

### 1.1 char_dict.json（汉字字典，支撑正向查找）

- **规模**：20,751 个汉字
- **结构**：`{ "字": { "tones": [...], "rhymes": { "韵书名": [韵部名, ...] } } }`

示例：

```json
"行": {
  "tones": ["平", "去"],
  "rhymes": {
    "Pingshuiyun": ["八庚", "二十四敬", "二十三漾", "七阳"],
    "Cilinzhengyun": ["第2部_平", "第2部_仄", "第11部_平", "第11部_仄"]
  }
}
```

**关键特征**：一个字可以属于**多个韵部**（约 3,500+ 字存在此情况）。原因有二：
- **多音字**：如"行"读 xíng 时属"八庚"（平），读 xìng/háng 时属其他韵部
- **声调分韵**：同一字的平声、仄声（上/去/入）归属不同韵部，如"中"平声属"一东"、去声属"一送"

### 1.2 rhyme_books.json（韵书，支撑逆向查找）

- **韵书数量**：2 部
  - `Pingshuiyun`（平水韵）：106 个韵部（平声 30 个，仄声 76 个）
  - `Cilinzhengyun`（词林正韵）：33 个韵部（平声 14 个，仄声 19 个）
- **结构**：每个韵部包含 `name`、`tone_type`（P=平/Z=仄）、`characters`（该韵部所有汉字列表）、`relations`（韵部间关系）

示例：

```json
"六麻": {
  "name": "六麻",
  "tone_type": "P",
  "characters": ["花", "家", "霞", "沙", "茶", ...],
  "relations": { "neighbor": ["五歌"] }
}
```

### 1.3 两份数据的关系

```
char_dict.json                          rhyme_books.json
┌──────────────┐                       ┌──────────────────────┐
│ "花" ──────────────正向查找──────────▷ │ Pingshuiyun: ["六麻"] │
│              │                       │ Cilinzhengyun: [...]  │
│              │                       └──────────────────────┘
│              │
│              │                       ┌──────────────────────┐
│              │◁──────逆向查找──────── │ "六麻".characters:    │
│              │                       │   ["花","家","霞",...] │
└──────────────┘                       └──────────────────────┘
```

两份数据是**互为镜像**的冗余关系：
- `char_dict["花"]["rhymes"]["Pingshuiyun"]` 包含 `"六麻"` ↔ `rhyme_books["Pingshuiyun"]["categories"]["六麻"]["characters"]` 包含 `"花"`
- 这意味着正向/逆向查找各自只需读取一份文件，**无需联表**。

## 2. 正向查找：字 → 韵部

### 2.1 场景

用户在创作画布中输入一个字（如"花"），需要立即知道：
- 这个字属于哪个（哪些）韵部？
- 这个字是平声还是仄声？
- 在当前选定的韵书下，这个字归属哪些韵部？

### 2.2 数据源

直接查询 `char_dict.json`，无需其他数据。

### 2.3 实现

```python
def lookup_char(char: str, book_name: str, char_dict: dict) -> dict:
    """
    正向查找：输入一个字，返回其在指定韵书下的所有韵部归属。
    
    Args:
        char: 单个汉字
        book_name: 韵书标识 ("Pingshuiyun" 或 "Cilinzhengyun")
        char_dict: 已加载的 char_dict 数据
    
    Returns:
        {
            "char": "花",
            "tones": ["平", "上"],
            "rhyme_categories": ["六麻"],          # 可能有多个
            "rhyme_categories_dedup": ["六麻"],     # 去重后（有些字同韵部出现两次）
        }
        若查无此字，返回 None
    """
    info = char_dict.get(char)
    if not info:
        return None
    
    categories = info.get("rhymes", {}).get(book_name, [])
    
    return {
        "char": char,
        "tones": info.get("tones", []),
        "rhyme_categories": categories,
        "rhyme_categories_dedup": list(dict.fromkeys(categories)),  # 保序去重
    }
```

### 2.4 多韵部的处理策略

一个字可能属于多个韵部，前端需区分处理：

| 场景 | 示例 | 策略 |
|------|------|------|
| 单韵部 | "花" 在平水韵只属 "六麻" | 直接显示 |
| 多韵部（同声调） | "枅" 平声，属 "一先" 和 "八齐" | 列出所有韵部，让用户选择 |
| 多韵部（跨声调） | "中" 平声属 "一东"、去声属 "一送" | 按 `tone_type` 分组展示（平/仄） |
| 重复韵部 | "东" 在词林正韵中 `["第1部_平", "第1部_平"]` | 去重后展示 |

### 2.5 API 设计

```
GET /api/char/lookup?char=花&book=Pingshuiyun
```

Response:
```json
{
  "char": "花",
  "tones": ["平", "上"],
  "rhyme_categories": [
    { "name": "六麻", "tone_type": "P" }
  ]
}
```

当一个字属于多个韵部时（如"行"）：
```json
{
  "char": "行",
  "tones": ["平", "去"],
  "rhyme_categories": [
    { "name": "八庚", "tone_type": "P" },
    { "name": "七阳", "tone_type": "P" },
    { "name": "二十四敬", "tone_type": "Z" },
    { "name": "二十三漾", "tone_type": "Z" }
  ]
}
```

## 3. 逆向查找：韵部 → 所有同韵字

### 3.1 场景

用户确定要押某个韵部（如"六麻"），需要：
- 查看该韵部下所有可用的汉字
- 按声调过滤（只看平声字 / 只看仄声字）
- 该韵部有哪些"邻韵"可以通融？

### 3.2 数据源

直接查询 `rhyme_books.json`，无需其他数据。邻韵等关系也在此文件中。

### 3.3 实现

```python
def lookup_category(book_name: str, category_name: str, 
                    rhyme_books: dict, char_dict: dict,
                    tone_filter: str = None) -> dict:
    """
    逆向查找：输入韵部名，返回该韵部下所有汉字。
    
    Args:
        book_name: 韵书标识
        category_name: 韵部名（如 "六麻"）
        rhyme_books: 已加载的 rhyme_books 数据 (RhymeBook 对象字典)
        char_dict: 已加载的 char_dict 数据（用于声调过滤）
        tone_filter: 可选，"P" 只返回平声字，"Z" 只返回仄声字
    
    Returns:
        {
            "category_name": "六麻",
            "tone_type": "P",
            "characters": ["花", "家", "霞", ...],
            "total": 158,
            "relations": { "neighbor": ["五歌"] }
        }
    """
    book = rhyme_books.get(book_name)
    if not book:
        return None
    
    category = book.categories.get(category_name)
    if not category:
        return None
    
    chars = list(category.characters)
    
    # 可选：按声调过滤
    if tone_filter and char_dict:
        filtered = []
        for c in chars:
            info = char_dict.get(c)
            if not info:
                continue
            tones = info.get("tones", [])
            # 判断是否匹配目标声调
            if tone_filter == "P" and any(t in ["平", "Ping", "ping", "yinping", "阴平", "yangping", "阳平"] for t in tones):
                filtered.append(c)
            elif tone_filter == "Z" and any(t in ["上", "去", "入"] for t in tones):
                filtered.append(c)
        chars = filtered
    
    return {
        "category_name": category.name,
        "tone_type": category.tone_type,
        "characters": chars,
        "total": len(chars),
        "relations": category.relations,
    }
```

### 3.4 邻韵扩展查询

在实际创作中，诗的首句允许邻韵（`neighbor` 关系），词可能涉及叶韵（`ye_ze`/`ye_ping`）。前端可以一次性请求"本韵部 + 所有关联韵部"的字：

```python
def lookup_category_with_related(book_name: str, category_name: str,
                                  rhyme_books: dict, char_dict: dict,
                                  include_relations: list = None) -> dict:
    """
    逆向查找（含关联韵部）。
    
    Args:
        include_relations: 要包含的关系类型列表，如 ["neighbor"]
    
    Returns:
        {
            "primary": { ... },          # 主韵部结果
            "related": [                  # 关联韵部结果
                { "relation": "neighbor", "category": { ... } },
            ]
        }
    """
    primary = lookup_category(book_name, category_name, rhyme_books, char_dict)
    if not primary:
        return None
    
    related = []
    if include_relations:
        for rel_type in include_relations:
            rel_names = rhyme_books[book_name].get_related(category_name, rel_type)
            for rel_name in rel_names:
                rel_result = lookup_category(book_name, rel_name, rhyme_books, char_dict)
                if rel_result:
                    related.append({
                        "relation": rel_type,
                        "category": rel_result
                    })
    
    return {
        "primary": primary,
        "related": related,
    }
```

## 4. 韵部列表查询

### 4.1 场景

用户手动选择韵部时，需要展示当前韵书下所有韵部的列表。

### 4.2 实现

```python
def list_categories(book_name: str, rhyme_books: dict, 
                    tone_filter: str = None) -> list:
    """
    列出指定韵书的所有韵部。
    
    Args:
        tone_filter: 可选，"P" 只返回平声韵部，"Z" 只返回仄声韵部
    
    Returns:
        [
            { "name": "一东", "tone_type": "P", "char_count": 182 },
            { "name": "二冬", "tone_type": "P", "char_count": 105 },
            ...
        ]
    """
    book = rhyme_books.get(book_name)
    if not book:
        return []
    
    result = []
    for cat_name, cat in book.categories.items():
        if tone_filter and cat.tone_type != tone_filter:
            continue
        result.append({
            "name": cat.name,
            "tone_type": cat.tone_type,
            "char_count": len(cat.characters),
        })
    
    return result
```

## 5. 前端集成要点

### 5.1 创作流程中的调用时机

```
用户输入韵脚字 "花"
       │
       ▼
正向查找 GET /api/char/lookup?char=花&book=Pingshuiyun
       │
       ▼
返回 rhyme_categories: ["六麻"]  ──只有一个──▷ 自动锁定韵部
       │                          ──有多个──▷ 弹出列表让用户选
       ▼
逆向查找 GET /api/rhyme/lookup?book=Pingshuiyun&category=六麻&include=neighbor
       │
       ▼
右侧栏展示 "六麻" 下所有同韵字（+ 邻韵字灰色展示）
```

### 5.2 性能考虑

| 操作 | 数据量 | 建议 |
|------|--------|------|
| 正向查找 (char → 韵部) | 返回 1~4 个韵部名 | 极轻量，无需缓存 |
| 逆向查找 (韵部 → 字) | 单韵部 60~850 字 | 首次请求后前端缓存该韵部 |
| 韵部列表 | 平水韵 106 项 / 词林正韵 33 项 | 页面加载时一次性获取并缓存 |
| char_dict 全量 | 20,751 条，约 3~4 MB | 可选方案：前端启动时全量加载到内存，后续正向查找零延迟（纯前端查） |

### 5.3 前端全量加载方案（可选优化）

若网络条件允许，可在页面初始化时一次性加载 `char_dict.json` 到前端内存：

```typescript
// 页面初始化
const charDict: Record<string, { tones: string[], rhymes: Record<string, string[]> }> = 
  await fetch('/api/static/char_dict.json').then(r => r.json());

// 之后所有正向查找都可以纯前端完成，零延迟
function lookupChar(char: string, bookName: string) {
  const info = charDict[char];
  if (!info) return null;
  const categories = [...new Set(info.rhymes[bookName] || [])];  // 保序去重
  return { char, tones: info.tones, categories };
}
```

这样只有逆向查找（韵部 → 字列表）需要请求后端，大幅减少网络开销。

## 6. 数据边界与注意事项

1. **去重**：`char_dict` 中同一字在同一韵书下可能出现重复韵部名（如 "东" 在词林正韵中 `["第1部_平", "第1部_平"]`），展示前务必去重。

2. **多韵部字的韵脚判定**：当一个多韵部字出现在韵脚位置时，checker 的 `_check_same_category` 方法采用"从宽"策略——只要该字的任意一个韵部与其他韵脚字存在交集，就判定为合韵。前端不需要强制用户选择"这个字到底读哪个音"。

3. **tone_type 与 tones 的区别**：
   - `tone_type` 是韵部级别的属性（`"P"` 或 `"Z"`），表示该韵部归属平声韵还是仄声韵
   - `tones` 是字级别的属性（如 `["平", "去"]`），表示该字可能的声调读法
   - 两者并不总是对应的，因为一个多音字可能既属于平声韵部又属于仄声韵部

4. **韵书间不互通**：平水韵的韵部名（如"六麻"）与词林正韵的韵部名（如"第10部_平"）是独立体系，不能混用。前端根据 `genre`（Shi/Ci）自动切换韵书。
